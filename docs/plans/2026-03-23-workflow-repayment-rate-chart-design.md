# 运营工作进度页「每个运营的回款率趋势图」调整文档

## 1. 调整背景

当前 `运营工作进度` 页面右侧概览卡片展示的是：

- 标题：`本期进度标记柱状图`
- 数据：当前筛选区间内，各流程标签被标记完成的次数

这个图更偏“流程动作统计”，不能直接反映每个运营接手店铺的回款表现。  
本次拟将它替换为：

- 标题：`每个运营的回款率趋势图`
- 图表类型：折线图(Line Chart)
- 维度：按日期展示、按运营分组

---

## 2. 本次回款率定义

### 2.1 推荐口径

由于当前数据库中**没有“应回款金额 / 目标回款金额”字段**，无法计算金额型回款率。  
因此本次建议采用**店铺回款率**口径：

```text
回款率 = 当日有正向回款的店铺数 ÷ 当日应统计店铺总数
```

### 2.2 公式拆解

对任意一天、任意一个运营，定义：

1. **当日有正向回款的店铺数**
   - 取当日 `daily_point_details`
   - 同一店铺当天所有抽点明细求和
   - 若该店铺当日净回款金额 `> 0`，记为 1 家
   - 最终按运营汇总去重店铺数

2. **当日应统计店铺总数**
   - 取 `shops` 中当前归属该运营的店铺
   - 满足：
     - `entryDate <= 当日`
   - 即：只要在当日已经录入，就计入分母，不排除 `已解约`、`无效店铺`

3. **回款率**
   - `回款率 = 有正向回款店铺数 / 应统计店铺总数`
   - 展示为百分比，例如 `37.5%`

> 说明：这是“店铺级回款覆盖率”，不是“金额完成率”。

- `开始日期`

### 3.3 运营筛选

继续沿用当前右上角运营切换：

- `全部运营`

当选择：

- `全部运营`：多条线同时展示

## 5. 接口调整方案

### 5.1 当前接口

当前概览图来自：

- `GET /api/workflow/summary`

当前返回结构中的图表数据为：

- `progressCountByItem`

### 5.2 拟新增返回字段

在 `WorkflowSummary` 中新增：

```ts
operatorRepaymentRateTrend: Array<{
  name: string;
  values: Array<{
    date: string;
    value: number;
  }>;
}>
```

其中：

- `name` = 运营名
- `date` = 日期
- `value` = 回款率（百分比数值，例如 `37.5`）

### 5.3 旧字段处理

本次是“替换图表”，不是新增第二张图。  
因此页面层不再使用 `progressCountByItem` 绘图。

但为了避免影响其他潜在逻辑，接口短期内可以先保留旧字段，页面只切换到新字段。

## 6. 数据计算流程

### 步骤 1：取日期区间

按页面传入的：

- `startDate`
- `endDate`

生成每日日期序列。

### 步骤 2：取有效店铺池

从 `shops` 中筛选：

- `entryDate <= 当前统计日`
- `shopStatus ∉ {已解约, 无效店铺}`

按运营聚合，得到当天每个运营的“应统计店铺总数”。

### 步骤 3：取每日抽点明细

从 `daily_point_details` 中取该区间所有数据：

- 同店铺同日先合并净回款金额
- 净回款金额 `> 0` 才算“当日有正向回款”

### 步骤 4：店铺归属运营

通过 `merchantId / storeId / shopName` 与 `shops` 关联，映射到运营。

### 步骤 5：计算回款率

对每个运营、每个日期：

```text
回款率 = 当日有正向回款店铺数 ÷ 当日应统计店铺总数 × 100
```

若分母为 0：

- 回款率按 `0` 处理

## 7. 边界与风险说明

### 7.1 这是店铺覆盖率，不是金额完成率

因为当前没有“应回款金额”字段，所以本次不能做：

- 实际回款金额 ÷ 应回款目标金额

本次做的是：

- “店铺当天是否有正向回款”的覆盖率趋势

### 7.2 运营归属使用当前主表归属

如果某些店铺历史上换过运营，本次仍按 `shops` 当前归属来统计。  
这与当前项目其他统计口径保持一致，但不等于严格的历史归属。

### 7.3 已解约与无效店铺处理方式

本次按你的最新要求：

- `已解约`
- `无效店铺`

都 **不从分母中排除**。

也就是说，当天回款率分母统计的是：

- 截至当日已录入、且归属该运营的全部店铺

不论它当前标签是：

- 正常
- 新店
- 无效店铺
- 已解约
